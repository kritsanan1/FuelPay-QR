# Master Prompt สำหรับระบบจ่ายน้ำมัน

## เป้าหมาย (Goals)
- สร้างระบบจ่ายน้ำมันที่สมบูรณ์แบบ
- รองรับการชำระเงินด้วย QR Code
- เชื่อมต่อ API กับ Payment Gateway เพื่อเช็คสถานะการชำระเงิน

## กระบวนการทำงาน (Workflow)
1. **เลือกจ่ายน้ำมัน**:  
   - ลูกค้าเลือกประเภทน้ำมันและปริมาณ หรือกำหนดจำนวนเงินที่ต้องการจ่าย
2. **สร้าง QR Code**:  
   - ระบบ generate QR Code สำหรับการชำระเงิน (Dynamic QR Code)
3. **ชำระเงิน**:  
   - ลูกค้าสแกน QR Code ผ่านแอปธนาคารหรือแอปที่รองรับ
4. **เช็คสถานะการชำระเงิน**:  
   - ระบบตรวจสอบสถานะการชำระเงินผ่าน API ของ Payment Gateway
5. **อนุญาตจ่ายน้ำมัน**:  
   - เมื่อชำระเงินสำเร็จ ระบบจะเปิดการจ่ายน้ำมันและแจ้งผลแก่ลูกค้า

## เครื่องมือที่ใช้ (Tools)
- **Frontend**: React + TailwindCSS + GSAP สำหรับ UI ที่ลื่นไหล
- **Backend**: Node.js + Express สำหรับการจัดการ API
- **Database**: MongoDB หรือ Firebase สำหรับการเก็บข้อมูลการชำระเงิน
- **Payment Gateway**: ใช้ API เช่น GB PrimePay, SCB Easy, หรือ KBank API

## ฟีเจอร์หลัก (Features)
- หน้า UI สำหรับเลือกประเภทน้ำมันและกำหนดจำนวนเงิน
- ระบบสร้าง QR Code สำหรับการชำระเงินแบบ Dynamic
- เชื่อมต่อ API เพื่อตรวจสอบสถานะการชำระเงินแบบ real-time
- ระบบแจ้งเตือนลูกค้าเมื่อชำระเงินสำเร็จ (ผ่านหน้าจอหรือ SMS)

## ข้อมูลที่ต้องใช้ (Data Requirements)
- ประเภทน้ำมัน (ชื่อ, ราคา/ลิตร)
- ปริมาณน้ำมันที่ต้องการ หรือจำนวนเงินที่ต้องการจ่าย
- สถานะการชำระเงิน (Pending, Success, Failed)

## การออกแบบ UI (UI Design)
- **โครงสร้าง**:  
   - ใช้ TailwindCSS ในการจัด layout ให้ responsive  
   - ใช้ GSAP แอนิเมต QR Code และสถานะการชำระเงิน  
- **องค์ประกอบ**:  
   - ปุ่มเลือกประเภทน้ำมัน  
   - Input กำหนดจำนวนเงิน  
   - QR Code พร้อมปุ่มสำหรับรีเฟรช  
   - แถบสถานะการชำระเงินที่เปลี่ยนสีตามผลลัพธ์  

## การเชื่อมต่อ API (API Integration)
1. **สร้าง QR Code**:  
   - Endpoint สำหรับ generate QR Code จาก Payment Gateway  
   - ส่งข้อมูลประเภทน้ำมันและจำนวนเงิน  
2. **ตรวจสอบการชำระเงิน**:  
   - Endpoint สำหรับ polling หรือ Webhook จาก Payment Gateway  
   - ระบบตรวจสอบสถานะการชำระเงินทุก 5 วินาที  

## ทดสอบและปรับปรุง (Testing & Iteration)
- ทดสอบการสร้าง QR Code และการสแกนในสถานการณ์จริง  
- ตรวจสอบความถูกต้องของข้อมูลที่ส่งไปยัง Payment Gateway  
- ปรับ UI ให้เหมาะกับการแสดงผลบนมือถือและหน้าจอใหญ่  

## จุดเด่น (Unique Selling Points)
- การจัดการสถานะการชำระเงินแบบ real-time  
- ระบบที่ออกแบบมาอย่าง responsive และ interactive  
- เชื่อมต่อกับ Payment Gateway ได้หลายเจ้า
